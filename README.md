# AI 리더스북 (AI CHECK-MATE 디지털 포트폴리오)

2학년 정보 수업의 두 수행평가 — **AI 시대 팩트 체크(CHECK)** 와 **AI 기반 웹앱 개발(MATE)** — 의
산출물과 사전/사후 설문을 학생별로 모아, 자신의 **성장**을 한눈에 보고 성취감을 느끼게 하는
디지털 포트폴리오 웹앱. 연구대회(중등수업혁신사례연구대회) 연구과제 3에 해당.

## 구조
```
etl/build_data.py   # 입력 데이터를 통합·계산 → web/data.js 생성
web/
  index.html        # 진입점
  styles.css        # 디자인 시스템 (다크, CHECK=파랑 / MATE=보라)
  app.js            # 로그인·포트폴리오·성장차트(SVG)·집계 대시보드. 외부 의존성 없음
  data.js           # (자동 생성) window.RB = {students, agg, meta}
.claude/launch.json # 미리보기용 정적 서버 (python http.server :8123)
```

## 실행
- 그냥 `web/index.html`을 더블클릭해도 열림(데이터는 `<script>`로 로드, 오프라인 동작).
- 또는 정적 서버: `python -m http.server 8123 --directory web`

## 데이터 재생성
입력 파일(학생 명단, 조별 피드백, 웹앱 목록, 설문 3종)을 갱신한 뒤:
```
python etl/build_data.py          # web/data.js 생성 (etl/enrich.json 있으면 병합)
python etl/build_link_report.py   # 연구대회\웹앱 링크 검증.xlsx 생성
```
`etl/enrich.json` = 드라이브 크롤 산출(워크플로우). 각 학생 웹앱 PDF 1페이지의
**임베드 하이퍼링크(최신 앱 링크)** + 팩트체크 활동지 문서 ID/링크. 앱은 이 최신 링크를 우선 사용.

## 입력 데이터 (로컬)
- 명단: `…\연구대회\학생 명단.xlsx` (반·학번·이름) — 모든 데이터의 기준 키(학번 4자리)
- 팩트체크(조별): `…\연구대회\AI 시대 팩트 체크 조별 피드백.xlsx` (팀·멤버·3개 점수·코멘트)
- 웹앱 목록: `…\연구대회\웹앱 목록.xlsx` (제목·설명·appUrl)
- 설문: 데스크탑의 사전 / 팩트체크 후 / 웹앱개발 후 (응답).xlsx

## 성장 측정 모델
- **성장(전→후)**: AI 원리·한계 이해 / 비판적 검증(CHECK) / 문제 발견·해결 / 프로그래밍 구현
- **도달 수준(사후)**: AI 협업(MATE) / 정보 신뢰성 판단 / 지속 동기
- 매핑 상세는 `etl/build_data.py`의 `CONSTRUCTS` 참조.

## 드라이브 보강 결과 (완료)
- 웹앱 PDF 1페이지 **임베드 링크** 216명 추출 → 앱은 PDF 최신 링크 사용(목록.xlsx 링크는 보관)
- `웹앱 링크 검증.xlsx`: 동일 100 / 변경됨 116 / 텍스트≠하이퍼링크 5건
- 팩트체크 활동지·웹앱 보고서(PDF)를 앱 내 모달(iframe `…/preview`)로 열람, 미리보기 차단 시 새 탭 링크 폴백
- 팩트체크는 조별 → 팀 대표 작성본을 팀원 전원에게 연결

## 남은 갭
- 웹앱 미제출/링크없음 소수(1명), 썸네일(현재 미생성) — 필요 시 추가
- 웹앱 점수(채점중)는 입력되면 ETL에 반영
